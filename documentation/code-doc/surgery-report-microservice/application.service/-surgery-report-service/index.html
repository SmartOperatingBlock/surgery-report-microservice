Application Service that has the objective of generating and storing the SurgeryReport of a surgicalProcess. It needs the patientVitalSigns, the consumedImplantableMedicalDevices, the medicalTechnologyUsage. It reaches the goal by orchestrating the surgeryReportRepository, the healthProfessionalRepository, the healthcareUserRepository and the roomRepository. If a SurgeryReport with the same SurgicalProcessID exists, then it returns null.
